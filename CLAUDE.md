# Claim Warriors -- Claude Code Hub

This Obsidian vault is the knowledge base for all AI-related work at Claim Warriors. Any agent or Claude Code session working on Claim Warriors AI should read this vault for context before doing work.

## Vault Structure

- **00-Company/** -- Business context. What Claim Warriors does, how the departments work, claim types, tools used. Read this first to understand the business
- **01-AI-Strategy/** -- Vision, roadmap, and decision principles for AI across the company
- **02-Projects/** -- Individual AI project specs. Each file is one project with status, requirements, dependencies, and open questions
- **03-Integrations/** -- MCP connectors and external tools. What's available, what it does, how it's used
- **04-Scripts/** -- AI-generated intake and communication scripts (organized by claim type + status)
- **05-Decisions/** -- Decision log. Why we chose X over Y, with rationale
- **06-Meeting-Notes/** -- Summaries and action items from Ben/Jo strategy calls

## How to Use This Vault

### Before Starting Any AI Work
1. Read `00-Company/about-claim-warriors.md` for business context
2. Read the relevant project file in `02-Projects/` for your specific task
3. Check `01-AI-Strategy/roadmap.md` to understand where the project fits in the overall plan
4. Check `03-Integrations/` for what MCP tools are available

### When Making Decisions
1. Check `01-AI-Strategy/principles.md` for decision-making guidelines
2. Log significant decisions in `05-Decisions/` with date, context, and rationale

### After Completing Work
1. Update the relevant project file status
2. Update `01-AI-Strategy/roadmap.md` checkboxes
3. If new meeting notes exist, add them to `06-Meeting-Notes/`
